## Цепочка 1: «Диалог → Знания» (Knowledge Pipeline)

**Цель:** превратить сырой звонок в работающее знание для следующих звонков.  
**Модули:** AdminDialogues → Агент → Библиотека → Скрипт → Кокпит

```
Новый диалог загружен в AdminDialogues
  ↓
AI очистка (убирает паузы, шум, посторонние реплики)
  ↓
AI анализ: ExtractedDialogueData
  — тип клиента, тип станка, возражения, как закрыто
  ↓
BatchInsights (из 5+ диалогов):
  — паттерны, портреты клиентов, топ-формулировки
  ↓
Агент (Knowledge Architect):
  — извлекает повторяющиеся успешные паттерны
  — создаёт черновики МП в стиле StyleDNA
  — находит пробелы (вопросы без экспертных ответов)
  — пополняет портреты клиентов
  ↓
Руководитель:
  — просматривает черновики МП → публикует/редактирует
  — обновляет этапы скрипта
  ↓
Следующий звонок:
  — Кокпит использует обновлённые МП
  — Тренажёр использует реальные паттерны для сценариев
```

**Ключевая связь:** Диалоги → `processed_scripts` → `batch_insights` → `micro_presentations` → `script_nodes`

**Статус цепочки:** 🔧 Частично (анализ диалогов есть, автоматическое создание МП агентом — запланировано)

## Цепочка 2: «Звонок → Сделка» (Sales Flow)

**Цель:** провести клиента от первого контакта до подписания договора.  
**Модули:** Битрикс24 → Кокпит → ERP → КП-генератор → Дожим

```
Входящий звонок (в Битрикс24)
  ↓
Менеджер открывает Кокпит параллельно звонку
  ↓
Этап: КВАЛИФИКАЦИЯ (Скрипт + Библиотека)
  — Открытые вопросы: «Для чего рассматриваете станок?»
  — Слоты: [Материал: Дуб], [Толщина: 40мм], [Объём: ⏳ждём данные]
  — Правая панель адаптируется по заполненным слотам в реальном времени
  — Мульти-задачи: вкладки [Задача 1: Резка] [Задача 2: Печать]
  — Авто-адаптация: ПВХ → предупреждение о рисках лазера
  — [⚠️ Нестандартная задача] → режим сбора для инженера
  ↓
Этап: ПРОСЧЁТ (ERP iFrame)
  — Скрипт подсказывает что уточнить перед просчётом
  — ERP открыт в правой части Кокпита
  — Менеджер собирает 2-3 варианта комплектации
  — Озвучивает цены + использует МП для обоснования
  ↓
Резюмирующая «Вилка»:
  «Итак: Вариант А — базовый за X, Вариант Б — с чиллером за Y. Верно?»
  ↓
КП-генератор:
  — Менеджер заполняет шаблон КП (данные из квалификации)
  — Генерация PDF → отправка клиенту (через Битрикс24)
  ↓
[пауза: клиент думает]
  ↓
Этап: ДОЖИМ (Кокпит → Скрипт Дожима)
  — Контекстная сводка сверху: «заложен чиллер 5200, бюджет впритык»
  — Методики компромиссов в правой панели
  — Аргументы против конкурентов
  — Снятие возражений → Сделка
  ↓
Договор:
  — ContractBuilder → заполнение → подпись/печать → PDF
```

**Ключевые переходы данных:**
- Квалификация → КП: данные о клиенте, тип станка, конфигурация
- Квалификация → Дожим: сохранённый контекст звонка

**Статус цепочки:** 🔧 Частично (Кокпит в разработке, ERP и Дожим — запланированы)

## Цепочка 3: «Шестерёнка → Улучшение» (Feedback Loop)

**Цель:** непрерывно улучшать систему через обратную связь от менеджеров.  
**Модули:** Кокпит/Тренажёр → Шестерёнка → Агент → Библиотека/Dev

```
Менеджер во время звонка или тренировки:
  — Столкнулся с пробелом в знаниях
  — Нажал ⚙️ → написал заметку
  — Автоконтекст: этап, тип станка, тип скрипта
  ↓
Агент (ежедневная рутина):
  — Классифицирует: script_gap / dev_task / unclear
  — Проверяет дубли и группирует похожие заметки
  — Для script_gap → черновик МП (⚠️ AI-черновик)
  — Для dev_task → готовый промт для Claude Code
  ↓
Руководитель:
  — Видит дайджест агента
  — Публикует МП → попадает в Библиотеку и Скрипт
  — Или копирует промт в Claude Code → разработчик реализует
  ↓
Менеджер:
  — Получает уведомление: «добавлена МП по вашему вопросу»
  — В следующем звонке/тренировке видит новую МП в Кокпите
```

**Почему это важно:** каждый пробел закрывается навсегда, не только для одного менеджера.

**Статус цепочки:** 💡 Запланирована полностью

## Цепочка 4: «Звонок → Развитие менеджера» (Quality & Growth)

**Цель:** превратить каждый реальный звонок в урок для менеджера.  
**Модули:** Контроль качества → Тренажёр → Аналитика → Руководитель

```
Реальный звонок завершён
  ↓
Транскрипт выгружен и загружен в систему (Контроль качества)
  ↓
Агент оценивает по чек-листу руководителя:
  — Забрал ли позицию эксперта?
  — Задавал ли открытые вопросы?
  — Использовал МП по критическим узлам?
  — Эмоциональный тон, не перебивал?
  ↓
Отчёт по этапам:
  Открытие: 9/10 ✅  Квалификация: 6/10 ⚠️  Возражения: 4/10 ❌
  Системная ошибка: «не уточнял объём производства»
  ↓
Руководитель:
  — Видит сводку по всем менеджерам
  — Точечно даёт обратную связь: «Иван, работай над Возражениями»
  — Назначает наряд: «пройди сценарий CO2 Эконом до пятницы»
  ↓
Менеджер → Тренажёр (по назначенному наряду)
  — Отрабатывает конкретно слабый этап
  — Шестерёнка фиксирует новые пробелы
  ↓
Прогресс в аналитике:
  — Сравнение: реальные звонки vs тренировки
  — Динамика по неделям
```

**Ключевой принцип:** оценка реальных звонков и тренировок по одному чек-листу → сопоставимые данные.

**Статус цепочки:** 💡 Запланирована полностью

## Цепочка 5: «Диалоги → Контент» (Content Pipeline)

**Цель:** превратить знания из диалогов в SEO-статьи для сайта.  
**Модули:** AdminDialogues → BatchInsights → AgentСтатьи → WordPress

```
BatchInsights из диалогов:
  — Топ боли клиентов: «часто спрашивают про резку капролона»
  — topPainPoints, articleTopicSuggestions
  ↓
Агент предлагает темы:
  «Много вопросов про капролон → 
   Статья: "Как резать капролон: 5 ошибок которые сожгут шпиндель"»
  ↓
ArticleGenerationWizard (4 шага):
  Тема → Черновик AI → Рестайл (StyleDNA Евгения) → Сохранить/Публикация
  ↓
WordPress публикация через REST API
```

**Важная связь StyleDNA:** статьи пишутся голосом Евгения — стиль извлекается из тех же диалогов которые загружены в систему.

**Статус цепочки:** ✅ Реализована (за исключением автоматических предложений тем от агента)

## Связи между цепочками

```
Цепочка 1 (Знания) питает Цепочку 2 (Звонок):
  Чем лучше библиотека → тем лучше скрипт → тем эффективнее звонок

Цепочка 3 (Шестерёнка) питает Цепочку 1 (Знания):
  Пробелы из реальных звонков → новые знания → обновлённый скрипт

Цепочка 4 (Развитие) питает Цепочку 2 (Звонок):
  Лучше обученный менеджер → более эффективный звонок

Цепочка 5 (Контент) питается от Цепочки 1:
  Те же диалоги → и в скрипт, и в статьи
```

## Будущие цепочки (долгосрочные)

### Цепочка 6: «Переписка → Квалификация» (Бот-Клон)
**Цель:** AI-бот ведёт первичную квалификацию в WhatsApp когда менеджеры заняты.
**Модули:** [[modules/19_bot_clone]] → [[modules/05_cockpit]] → [[modules/14_bitrix24]]

```
WhatsApp: клиент пишет → Бот-Клон отвечает голосом Евгения
  ↓
Бот задаёт квалификационные вопросы (те же слоты что Кокпит)
  ↓
При сложном вопросе → ставит метку → менеджер подхватывает
При готовности → записывает на звонок с менеджером
  ↓
Вся переписка → Firestore → Кокпит видит контекст до звонка
```

### Цепочка 7: «Академия → Компетенции» (Обучение)
**Цель:** менеджер растёт через видеотеку + тренажёр, не только через реальные звонки.
**Модули:** [[modules/18_academy]] → [[modules/10_trainer]] → [[modules/11_analytics]]

```
Академия: видео по продукту с RAG-поиском
  ↓ (менеджер ищет ответ во время переписки)
Тренажёр: отрабатывает знание с AI-клиентом
  ↓
Реальный звонок: применяет — Контроль качества оценивает
  ↓
Пробел → Шестерёнка → Агент → новое видео/МП в Академию
```
