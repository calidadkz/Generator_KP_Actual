## Роли

### Руководитель (Администратор)
Полный доступ ко всему.

✅ Admin Panel (все вкладки)  
✅ Агент-администратор (чат + действия в БД)  
✅ Аналитика всех менеджеров  
✅ Настройки рутин агента  
✅ Журнал действий агента  
✅ Дашборд расходов API  
✅ Создание/редактирование шаблонов КП и Договоров  
✅ Manager Cockpit (может зайти)  
✅ Тренажёр (может зайти)  

### Менеджер
Только рабочие инструменты.

✅ Manager Cockpit (скрипт + МП во время звонка)  
✅ Тренажёр (запуск сессий, задания)  
✅ Своя аналитика + сравнительная (обезличенная по коллегам)  
✅ Шестерёнка (заметки из звонков/тренировок)  
✅ КП-генератор (только заполнять шаблоны)  
✅ Договора (только заполнять)  
❌ Admin Panel  
❌ Агент  
❌ Настройки рутин  
❌ Дашборд расходов API  
❌ TemplateMapper (создание шаблонов)

## Разграничение в КП и Договорах

| Действие | Менеджер | Руководитель |
|----------|----------|-------------|
| Заполнить шаблон → PDF | ✅ | ✅ |
| Создать/редактировать шаблон | ❌ | ✅ |
| Удалить шаблон | ❌ | ✅ |

Агент будет помогать руководителю при создании шаблонов (детали — в [[modules/03_kp_generator]]).

## Тренажёр: два режима доступа

1. **Свободная тренировка** — менеджер сам выбирает сценарий из опубликованных
2. **Наряд-задание** — руководитель назначает набор сценариев конкретному менеджеру с дедлайном

```
Сущность training_assignment:
{
  managerId, 
  scenarios: [scenarioId1, scenarioId2...],
  deadline,
  status: "pending" | "in_progress" | "completed",
  completedAt,
  agentReport
}
```

## Техническая реализация

Firebase Authentication (уже используется в проекте через Firebase SDK).

```
Руководитель: email + пароль (evgeny@calidad.kz)
Менеджеры:    email + пароль (ivan@calidad.kz, ...)
```

При входе → Firebase возвращает role → React показывает разрешённые разделы → Firestore Security Rules блокируют на уровне БД.
