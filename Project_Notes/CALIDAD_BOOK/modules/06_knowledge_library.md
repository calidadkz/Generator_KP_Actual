## Динамические переменные ERP в атомах знаний

Атом (МП) может ссылаться на конкретную позицию ERP вместо жёстко прописанной цены.

**Синтаксис переменных:**
```
{ERP_Option_Price_ID_55}   →  актуальная цена опции #55 из ERP
{ERP_Model_Name_ID_55}     →  название модели #55
{ERP_Stock_ID_55}          →  наличие/отсутствие на складе
```

**Пример атома с переменной:**
```
МЕТОДОЛОГИЯ: Аргумент за чиллер
«С чиллером CW-5200 трубка живёт в 3 раза дольше.
 Стоимость добавки: {ERP_Option_Price_ID_55} — окупается за 6 месяцев.»
```

**При показе в Кокпите:** `{ERP_Option_Price_ID_55}` → `145 000 ₸` (из актуального ERP).

**Преимущество:** руководитель пишет аргумент один раз, цена обновляется автоматически при смене прайса в ERP. Нет устаревших цен в МП.

**Реализация:** синхронизация ERP → Firestore (коллекция `erp_price_cache`) раз в N часов. Подробно: [[modules/13_erp_integration]].
