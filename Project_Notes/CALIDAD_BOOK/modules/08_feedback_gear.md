## Суть системы

Это не просто кнопка для заметок. Это **организационный механизм постоянного улучшения** — как продаж (знания, скрипт), так и самого приложения (функционал).

Менеджер — ключевой источник знаний о том, чего не хватает в системе. Он единственный кто работает с реальными клиентами и видит живые пробелы.

**Из документа (дословно):**
> «Не это не просто шестерёнка расположенное кнопка — так чтоб менеджер пока помнит что-то сразу зафиксировал туда.»

---

## Два потока в одной системе

Менеджер не выбирает поток — просто пишет текст. Агент классифицирует.

| Поток | Что означает | Что происходит |
|-------|-------------|----------------|
| **Знания** | «Нет ответа в скрипте/библиотеке» | Агент предлагает черновик МП или правку скрипта |
| **Функционал** | «Нужна доработка программы» | Агент формулирует готовый промт для Claude Code |

## Полный процесс

```
━━━ МЕНЕДЖЕР (Кокпит или Тренажёр) ━━━

Нажимает ⚙️ → открывается форма:
  Текст: "Клиент спросил можно ли резать пенопласт"
  [Скрин — опционально]
  [🔴 Срочно — если нужен ответ немедленно]
  [Только для руководителя — если личное]
  
Система автоматически добавляет контекст:
  имя менеджера, тип скрипта (Квалификация/Дожим),
  текущий этап, тип станка

Статус заметки: "новая"

━━━ АГЕНТ (рутина ежедневно или мгновенно для 🔴 Срочно) ━━━

Шаг 1 — Классификация каждой новой заметки:
  script_gap    → нет ответа на вопрос клиента в скрипте
  knowledge_gap → нет технической информации в библиотеке
  dev_task      → нужна доработка функционала программы
  unclear       → непонятно без уточнений
  (заметка может получить несколько типов одновременно)

Шаг 2 — Проверка дублей:
  → есть похожая заметка от другого менеджера?
    → пометить как группа, показать контекст обеих
  → есть похожий контент уже в базе знаний?
    → отметить "возможно уже решено"

Шаг 3 — Подготовка предложения:
  script_gap    → черновик МП (⚠️ AI-черновик, непроверено)
  dev_task      → готовый промт для Claude Code
  unclear       → список уточняющих вопросов

Шаг 4 — Дайджест для руководителя:
  "Обработано 7 заметок за 24 часа:
   → 3 script_gap: черновики МП ждут просмотра
   → 2 dev_task: промты сформулированы
   → 1 группа: Иван и Марина про одно и то же (380В) — объединить?
   → 1 unclear: скрин без текста, нужна расшифровка"

━━━ РУКОВОДИТЕЛЬ (Admin Panel → вкладка «Обратная связь») ━━━

Видит дайджест → разбирает:

script_gap:
  [Просмотреть черновик МП]
  Рядом показаны 3 похожих МП уже в базе (для сравнения)
  → [Опубликовать] [Редактировать] [Отклонить]
  После публикации → уведомление автору заметки

dev_task:
  [Скопировать промт для Claude Code]
  → [Взять в работу] [Отклонить]

группа:
  Видит обе заметки с их контекстом (кто, какой звонок, какой тип станка)
  → [Объединить → ответ идёт обоим] [Создать 2 отдельных МП]

unclear:
  [Написать менеджеру уточняющий вопрос]
  Менеджер получает уведомление → отвечает

Статусы: Исполнено / Отклонено / В работе (+ дата)

━━━ МЕНЕДЖЕР (видит результат) ━━━

Колокольчик 🔔: "Ваш вопрос про пенопласт — добавлена МП в раздел Материалы"
Вкладка «Мои заметки»: статус обновился, комментарий руководителя
```

## Интерфейс

### Флоатинг-кнопка ⚙️ (во время звонка/тренировки)
- Всегда видна, не мешает основному интерфейсу
- Клик → компактная форма (текст + скрин + флаги)
- Отправка за 5 секунд

### Вкладка «Мои заметки» (отдельный раздел меню)
**Менеджер видит:**
```
МОИ ЗАМЕТКИ        ЗАМЕТКИ КОЛЛЕГ (обезличено)
──────────────     ─────────────────────────────
"Вопрос про пенопласт"  "Про 380В" — Иван
Статус: ✅ Решено        Статус: 🔄 В работе
Ответ: добавлена МП      
```

**Руководитель видит (Admin Panel → «Обратная связь»):**
```
Дайджест агента    Все заметки        Статистика
───────────────    ──────────────     ────────────
3 ждут просмотра   [фильтры]          43 за месяц
2 промта готовы                       88% обработано
1 объединить?                         12 → в МП
```

## Подводные камни и решения

| # | Риск | Решение |
|---|------|---------|
| 1 | Агент неверно классифицировал | Двухшаговое: сначала «правильно?» → потом предложение |
| 2 | Черновики копятся непросмотренными | Лимит 10 черновиков → иначе только уведомление «разбери» |
| 3 | Похожие заметки — разный контекст | Агент показывает контекст каждой перед объединением |
| 4 | Скрин без текста | Auto-unclear, агент не интерпретирует скрин самостоятельно |
| 5 | dev_task завис без движения | Алерт через 30 дней без обновления статуса |
| 6 | Руководитель — бутылочное горлышко | Быстрое одобрение × N для похожих паттернов |
| 7 | Ошибочная МП распространяется | Маркировка ⚠️ AI-черновик до явного одобрения Евгением |
| 8 | Менеджер не знает что вопрос решён | Уведомление автору когда его МП опубликована |
| 9 | Одна заметка = два типа задач | Заметка может иметь несколько классификаций параллельно |
| 10 | Нет метрики работает ли система | Дашборд самой системы: создано/обработано/в МП/ожидают |
| 11 | Менеджер не пишет личные трудности | Опция [Только для руководителя] |
| 12 | Срочный пробел прямо во время звонка | Флаг 🔴 Срочно → мгновенное уведомление, не ждёт рутины |

---

## Критически важно: маркировка AI-контента

Все МП созданные агентом из заметок помечаются `⚠️ AI-черновик` до явного подтверждения Евгением.

В скрипте менеджер видит такую МП как «непроверенная» — может использовать, но знает что это черновик. После того как Евгений проверил → снимается метка.

Рутина агента: «покажи все МП старше 60 дней с меткой AI-черновик».
