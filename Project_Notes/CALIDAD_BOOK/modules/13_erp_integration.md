## Концепция

ERP (собственная система, есть веб-интерфейс/URL) встраивается через iFrame прямо в Кокпит менеджера.

**Цель:** всё в одном окне — скрипт подсказывает «что предложить», менеджер собирает сборку в ERP.

---

## Процесс (из документа)

```
Менеджер доходит до этапа просчёта →
Скрипт предоставил: что уточнено у клиента, какие варианты рассматривать
→ Менеджер открывает ERP (iFrame в правой части экрана)
→ Собирает 2-3 варианта комплектации
→ Озвучивает цены из ERP
→ Использует МП из скрипта для обоснования цены
→ Скрипт формирует «Резюмирующую Вилку»: 2-3 варианта с ценами
→ Менеджер зачитывает для подтверждения («Верно?»)
```

---

## Связь скрипта с ERP-ассортиментом

Скрипт знает ассортимент ERP → подсказывает варианты даже если менеджер забыл.
При добавлении новых сборок в ERP → скрипт адаптируется гибко.

**Динамические цены в МП (запланировано):**
Вместо «Чиллер стоит 200к» → `{ERP_Price_CW5200}` — система подтягивает актуальную цену.
